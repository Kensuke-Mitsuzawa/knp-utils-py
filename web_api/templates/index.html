Web api documentation

run_knp_api

API type: POST

API end pint: run_knp_api

What you can do

With this api, you can start knp-utils process.

This api is job queue. So, you get task-id as return-value.

With the task-id, you can ask status api

Post data scheme

Json data model is

{"is_use_jumanpp": bool, "n_jobs": int,  "input_document": {"text-id": string, "text": string}}

status api

API type: GET

API end pint: status/[task-id]

What you can do

You're able to know status of task which you started.

Example

status/1cebe42a-235e-4cb1-a8dd-9c3105c47f63

Response

{ "started_at": string, "state": string, "status": string" }
Example
{
  "started_at": "2017-02-25 12:16:43",
  "state": "SUCCESS",
  "status": "completed"
}

To test the api with curl

curl localhost:5000/status/1cebe42a-235e-4cb1-a8dd-9c3105c47f63 -H "Content-type: application/json" -X GET
